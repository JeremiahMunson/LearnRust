Common Programming Concepts

Variables and Mutability

To declare a variable we use the keyword let. This creates a new, immutable variable which means the value of the variable cannot be changed. This is not the same as const which declares a constant. A constant must be annotated (type explicity written) while variables usually don't need to. Constants can be declared in any scope, including the global scope, making them useful for values that many parts of code need to know about. Constants must also be set only to a constant expression whereas a variable can be set as the result of a function call or any other value that can only be computed at runtime. You can declare a mutable variable by doing let mut, but you can't make an immutable variable mutable.

You can shadow a variable by declaring a variable with the same name. This allows you to change the value and even the type of a variable you have already declared. For example, let's say you declare a variable let x = 5; and now you want a variable called x that holds a string. If you don't need the current variable, you can shadow it by declaring the variable again (let mut x = String::new();).
